Description
The Gen12 Service Pack for ProLiant (SPP) is a comprehensive system software and firmware update solution, which is delivered as a single
ISO image. It uses the Smart Update Manager (SUM) as the deployment tool and is tested on all the HPE ProLiant Gen12 servers as defined
in the Server Support Guide.

Customizing the SPP download
About this task
Customizing the SPP download can be done using the following steps:
Procedure
1. Select the appropriate SPP.
2. Click Create a new bundle tile (plus sign tile).
3. Complete the Configure Your Bundle step with the applicable information of your choice and Click Next.
4. Select the Filters button.
To download only the supplements, deselect the SPP and ensure that only the required supplement are selected.
5. Select filters as needed.
No selection in any given category means everything in that category will be included.
6. Click Build button and preview the selected components.
7. Click Submit Build button.
Upon submission of the build request, it will be queued and processed later.
Results
On the successful completion of the build, it will be notified through an email notification or Indicated on the new tile.

Installation
There are two methods for using the Service Pack for ProLiant to update the ProLiant servers. The Smart Update Manager (SUM) tool is
used to deploy the updates.
There are different deployment modes:
Online mode
Offline mode
These modes can be Local or Remote.
Subtopics
Deployment modes
Deployment modes
Online mode
1. Local host update: It runs on a Linux hosted operating system.
a. Interactive mode - Follow the onscreen steps to update the firmware components.
To initiate deployment in the interactive mode, run the launch_sum.sh script which is found in the root of the .iso image.
b. Automatic mode – The drivers, software, and firmware components will be updated without any user interaction.
2. Remote Update: SUM runs on one system (running Windows or Linux operating system), but updates other physical nodes.
a. Interactive mode - Follow the onscreen steps to update the firmware components.
To initiate deployment in the interactive mode, run the launch_sum.bat (windows) or launch_sum.sh (Linux) script

Offline mode
Offline mode: The server is booted to the .iso image.
1. Interactive mode - Follow the onscreen steps to update the firmware components.
2. Automatic mode - Automatic mode will by default run the firmware update without any user interaction, after sitting for 10 seconds at
the menu screen when an SPP .iso image is used to boot the server.

Deployment in offline mode
About this task
To initiate the deployment in offline mode:
Procedure
1. Boot the server to the SPP using one of the supported methods.
Mounting the ISO or using a bootable USB key.
2. Select either the Automatic or Interactive mode.
a. If the Automatic mode is selected, then the firmware will be automatically updated on the server without any further interaction.
b. If the Interactive mode is selected, follow the instructions on the screen.
3. Select the Firmware Update option on the screen to start SUM.
